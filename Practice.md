# Практика по курсу Python

## О файле

Этот файл создан как практическое дополнение к лекциям курса. Его задача - помочь не просто прочитать теорию, а закрепить каждую тему через систему упражнений, выстроенных от простого к более сложному.

Практика в этом файле организована по лекциям, и для каждой темы предусмотрены упражнения, которые помогут вам не только понять материал, но и научиться применять его на практике. При этом задачи отсортированы по уровню сложности и типу навыков, которые они развивают. Это позволяет вам постепенно наращивать свои знания и умения, не перегружая себя сразу сложными задачами.

Такой подход помогает одновременно решать несколько задач:
- Закрепление теории через практические упражнения.
- видеть постепенный рост сложности;
- тренировать как базовый синтаксис, так и прикладное мышление;
- постепенно готовиться к заданиям уровня `junior Python developer`.

> Этот файл не заменяет лекции, а работает вместе с ними. Сначала рекомендуется изучить теорию, повторить примеры из лекции, а уже затем переходить к практическим задачам.

---

## Как устроена практика

Каждая лекция оформляется как отдельный блок. Внутри блока будет:

- краткое описание того, что должен уметь студент после темы;
- список навыков, которые тренируются в этом разделе;
- задачи, отсортированные по сложности;
- в некоторых темах - мини-практика или задания, приближённые к формату `junior-собеседований`.

Практика будет строиться не как случайный набор упражнений, а как последовательная система:

- сначала простые задачи на понимание синтаксиса;
- затем задачи на комбинирование нескольких конструкций;
- затем более прикладные упражнения;
- затем задачи повышенного уровня, где нужно подумать самостоятельно.

## Уровни сложности будут обозначены так:

Для удобства все задачи будут размечены по сложности.

### `★` - Базовый уровень

Задачи на закрепление одной идеи или конструкции. Обычно это первые упражнения после изучения темы.

### `★★` - Уверенный уровень

Задачи, где уже нужно немного комбинировать знания внутри одной темы.

### `★★★` - Средний уровень

Задачи, в которых сочетаются несколько конструкций или требуется чуть больше самостоятельного анализа.

### `★★★★` - Продвинутый уровень / junior-собеседования 

Более серьёзные задачи, приближённые к прикладной разработке, тестовым заданиям или популярным вопросам для начинающих разработчиков.

## Типы задач внутри каждой лекции

Чтобы практика была действительно полезной, внутри каждого блока задачи будут разделяться по своему назначению.

### Разогрев

Небольшие упражнения на быстрое закрепление синтаксиса и основных конструкций.

### Основные задачи

Базовый набор задач по теме, который обязателен для уверенного понимания материала.

### Усложнённые задачи

Задания, где уже нужно объединять несколько идей, аккуратно продумывать решение и избегать типичных ошибок.

### Практика junior-уровня

Задачи, которые часто встречаются на собеседованиях для начинающих разработчиков. Они требуют не только знания синтаксиса, но и умения применять его в нестандартных ситуациях.

Подготовьте небольшой учебный проект и разбейте изменения на три логичных коммита с понятными сообщениями.

---

### Задача 23. Hotfix-ветка
**Уровень:** `★★★★`  
**Теги:** `branch`, `git merge`, `workflow`

**Описание:**  
Смоделируйте ситуацию, когда в основном проекте найден баг:

- создайте ветку `hotfix/login-bug`;
- исправьте проблему;
- сделайте коммит;
- влейте исправление обратно в основную ветку.

---

### Задача 24. Удаление временного файла из отслеживания
**Уровень:** `★★★★`  
**Теги:** `.gitignore`, `git reset`, `repo`

**Описание:**  
Представьте, что вы случайно добавили временный файл в репозиторий.
Уберите его из отслеживания и настройте `.gitignore`, чтобы он больше не появлялся в `git status`.

---

### Задача 25. Выборочный перенос исправления
**Уровень:** `★★★★`  
**Теги:** `git cherry-pick`, `bugfix`

**Описание:**  
Есть ветка `experimental`, в которой много изменений, но вам нужно забрать в основную ветку только один commit с исправлением бага.
Решите задачу через `git cherry-pick`.

---

### Задача 26. Подготовка чистой истории перед сдачей
**Уровень:** `★★★★`  
**Теги:** `git log`, `branch`, `merge`, `rebase`

**Описание:**  
Подготовьте учебный репозиторий к сдаче:

- создайте ветку с задачей;
- сделайте 2-3 осмысленных коммита;

- покажите итоговую историю через `git log`.

# Лекция 10: Git. Удалённый репозиторий.

> Лекция: [lesson10.md](lesson10.md)

## Что вы освоите после этой темы

После изучения этой темы студент должен понимать, что такое удалённый репозиторий, чем отличаются публичные и приватные репозитории, когда использовать `HTTPS` и `SSH`, как клонировать репозиторий, подключать `remote`, отправлять и получать изменения, работать с удалёнными ветками и создавать `Pull Request`.

## Какие навыки тренируются

`git`, `remote`, `git clone`, `git remote`, `git push`, `git pull`, `HTTPS`, `SSH`, `GitHub`, `GitLab`, `Bitbucket`, `Azure DevOps`, `public`, `private`, `Pull Request`, `collaborator`

## Разогрев

### Задача 1. Выбор платформы
**Уровень:** `★`  
**Теги:** `GitHub`, `GitLab`, `Bitbucket`

**Описание:**  
Назовите минимум три платформы для удалённых репозиториев и выберите одну, на которой вы будете работать в курсе.

---

### Задача 2. Публичный или приватный
**Уровень:** `★`  
**Теги:** `public`, `private`

- коммерческий проект с закрытым кодом.

### Задача 3. HTTPS-ссылка репозитория
**Уровень:** `★`

### Задача 4. SSH-ссылка репозитория
**Уровень:** `★`  
**Теги:** `SSH`, `remote`

### Задача 5. Клонирование репозитория
